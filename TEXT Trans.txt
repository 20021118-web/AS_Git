1) 기존 흔적 제거 (없으면 "Can't open service!" 떠도 무시)
C:\ServerBox\install\nssm.exe remove ServerBox-Parts confirm
2) 부품 서버 서비스 등록
C:\ServerBox\install\nssm.exe install ServerBox-Parts C:\ServerBox\venvs\parts\Scripts\python.exe "-m uvicorn main:app --host 127.0.0.1 --port 8001"
3) 작업 폴더 지정 (main.py·static 경로 기준)
C:\ServerBox\install\nssm.exe set ServerBox-Parts AppDirectory C:\ServerBox\apps\parts
4) 자동 시작 설정
C:\ServerBox\install\nssm.exe set ServerBox-Parts Start SERVICE_AUTO_START
5) 서비스 시작
C:\ServerBox\install\nssm.exe start ServerBox-Parts
6) 부품 서버 단독 확인 (한 줄)
Start-Sleep 3; Get-Service ServerBox-Parts; (Invoke-WebRequest http://localhost:8001/ -UseBasicParsing -TimeoutSec 8).StatusCode
→ Running + 200 이면 서버 엔진 정상. (안 되면 여기서 멈추고 알려주세요)

7) Caddy에 /parts 길 추가 — 이 한 줄 (기존 /web도 함께 유지)
Set-Content C:\ServerBox\gateway\Caddyfile ":8080 {`n`thandle_path /web/* {`n`t`troot * ../apps/webapp`n`t`tfile_server`n`t}`n`tredir /web /web/ 308`n`thandle_path /parts/* {`n`t`treverse_proxy 127.0.0.1:8001`n`t}`n`tredir /parts /parts/ 308`n}" -Encoding ascii
8) 게이트웨이 재시작
Restart-Service ServerBox-Gateway
9) 최종 확인 (한 줄) — /web 과 /parts 둘 다 200 이어야 함
Start-Sleep 2; "web=" + (Invoke-WebRequest http://localhost:8080/web/ -UseBasicParsing -TimeoutSec 8).StatusCode; "parts=" + (Invoke-WebRequest http://localhost:8080/parts/ -UseBasicParsing -TimeoutSec 8).StatusCode
→ web=200 + parts=200 이면 완료!

완료 후 접속 주소
주소	계산 위치	용도
http://10.232.4.196:8080/parts/	서버 PC	업무 PC 가볍게 (권장)
http://10.232.4.196:8080/web/	접속한 PC 브라우저	기존 방식 (그대로 유지됨)
/parts/ 로 접속해서 매칭을 돌리면, 이제 서버 PC 작업 관리자에 python 프로세스가 CPU/RAM을 쓰는 게 보입니다. (SIFT 매칭·배경 제거를 서버가 계산)

확인 팁
매칭 중 서버 PC 작업 관리자 → 세부정보 탭 → python.exe CPU/메모리 올라가는지
재부팅 후에도 /parts/ 열리면 자동 시작 정상
AI(CLIP) 유사도는 이 단계엔 없고 원조 SIFT로 매칭합니다(8GB에 적합). 나중에 램 늘리면 AI 서버도 추가 가능
9번 결과(web= / parts= 값)를 알려주시면 팀원 공유 안내문을 /parts 기준으로 정리해 드리겠습니다.
